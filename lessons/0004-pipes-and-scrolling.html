Lesson 4: Pipes & Scrolling

Making obstacles that scroll from right to left.

New concept: Arrays and Objects

So far we've used simple variables: one birdY, one birdVelocity. But pipes are different -- there will be many of them on screen at once, and each one needs its own x-position and gap position. We need two new tools:

An object groups related values together. Think of it as a labeled box: 

const pipe = { x: 400, gapY: 200 };

pipe.x gives 400. pipe.gapY gives 200.

An array is an ordered list. We'll keep all our pipes in an array: 

const pipes = [];                          // empty list
pipes.push({ x: 400, gapY: 180 });        // add a pipe to the list
pipes.push({ x: 550, gapY: 250 });        // add another
// pipes is now: [{x:400, gapY:180}, {x:550, gapY:250}]
push() adds to the end, splice() removes. When a pipe scrolls off the left edge, we remove it from the array. When we need a new pipe, we push it.

The pipe lifecycle

Spawn

At the right edge: x = canvas.width. Random gap position.

Scroll

Each frame: pipe.x -= 2. Moves left.

Remove

When x < -pipeWidth: splice it out.

Drawing a pipe pair

Each pipe is actually two rectangles: one above the gap and one below. Here's how we draw one pipe object: 

const pipeWidth = 60;
const gapSize = 140;
const groundY = canvas.height - 40;

function drawPipe(pipe) {
  ctx.fillStyle = '#27ae60';

  // Top pipe: from the top of the canvas down to the gap
  ctx.fillRect(pipe.x, 0, pipeWidth, pipe.gapY);

  // Bottom pipe: from the gap bottom down to the ground
  const bottomPipeTop = pipe.gapY + gapSize;
  ctx.fillRect(pipe.x, bottomPipeTop, pipeWidth, groundY - bottomPipeTop);
}

The full code so far

const canvas = document.getElementById('game');
const ctx = canvas.getContext('2d');
const groundY = canvas.height - 40;
const pipeWidth = 60;
const gapSize = 140;
const pipeSpeed = 2;
const spawnInterval = 120;  // frames between new pipes

let birdY = 250;
let birdVelocity = 0;
const gravity = 0.4;
const flapStrength = -7;
let pipes = [];
let frameCount = 0;

function spawnPipe() {
  const minGap = 50;
  const maxGap = groundY - gapSize - 50;
  const gapY = minGap + Math.random() * (maxGap - minGap);
  pipes.push({ x: canvas.width, gapY: gapY });
}

function update() {
  birdVelocity += gravity;
  birdY += birdVelocity;
  frameCount++;

  // Move pipes left
  for (const pipe of pipes) {
    pipe.x -= pipeSpeed;
  }

  // Remove off-screen pipes
  pipes = pipes.filter(pipe => pipe.x > -pipeWidth);

  // Spawn new pipes
  if (frameCount % spawnInterval === 0) {
    spawnPipe();
  }

  requestAnimationFrame(update);
}

function draw() {
  ctx.clearRect(0, 0, canvas.width, canvas.height);

  // Sky
  ctx.fillStyle = '#a8d8ea';
  ctx.fillRect(0, 0, canvas.width, canvas.height);

  // Pipes
  for (const pipe of pipes) {
    ctx.fillStyle = '#27ae60';
    ctx.fillRect(pipe.x, 0, pipeWidth, pipe.gapY);
    ctx.fillRect(pipe.x, pipe.gapY + gapSize, pipeWidth, groundY - pipe.gapY - gapSize);
  }

  // Ground
  ctx.fillStyle = '#8d6e63';
  ctx.fillRect(0, groundY, canvas.width, 40);

  // Bird
  ctx.fillStyle = '#f1c40f';
  ctx.fillRect(180, birdY, 40, 40);

  requestAnimationFrame(draw);
}
New syntax alert: for (const pipe of pipes) loops through every pipe in the array. pipes.filter(pipe => pipe.x > -pipeWidth) creates a new array keeping only pipes that are still on screen. Both are standard JavaScript patterns -- you'll use them constantly.

  • Open the console and type pipes.length while playing to see how many pipes are active

Key takeaways

  • Objects { x: 400, gapY: 200 } group related data together
  • Arrays [...] hold lists of things -- perfect for multiple pipes
  • push() adds to an array, splice()/filter() removes
  • Math.random() generates a random number between 0 and 1 for varied gap positions
  • for (const pipe of pipes) loops through every pipe to update or draw it
  • The game now looks nearly complete -- just missing collision detection and scoring
