Lesson 3: User Input & Flapping

Now you control the bird.

Listening for a key press

The browser fires an event every time the user presses a key. We can "listen" for that event and run our own code when it happens. The event is called keydown. 

document.addEventListener('keydown', function(event) {
  if (event.code === 'Space') {
    birdVelocity = -7;   // push the bird upward
  }
});

Let's unpack this:

  • addEventListener('keydown', ...) -- "browser, tell me when any key is pressed"
  • The second argument is a function -- a block of code that runs when the event fires
  • event.code tells us which key was pressed ('Space', 'ArrowUp', etc.)
  • We set birdVelocity = -7 -- a negative velocity means the bird moves upward
Why -7? The sign controls direction: negative = up, positive = down. The number controls how hard the flap is. 7 pixels per frame feels responsive. Gravity (0.4 per frame) will eventually pull the bird back down.

Putting it together

let birdY = 300;
let birdVelocity = 0;
const gravity = 0.4;
const flapStrength = -7;

function update() {
  birdVelocity += gravity;
  birdY += birdVelocity;
  requestAnimationFrame(update);
}

function draw() {
  ctx.clearRect(0, 0, canvas.width, canvas.height);

  ctx.fillStyle = '#a8d8ea';
  ctx.fillRect(0, 0, canvas.width, canvas.height);

  ctx.fillStyle = '#f1c40f';
  ctx.fillRect(180, birdY, 40, 40);

  requestAnimationFrame(draw);
}

document.addEventListener('keydown', function(event) {
  if (event.code === 'Space') {
    birdVelocity = flapStrength;   // shoot upward!
  }
});

requestAnimationFrame(update);
requestAnimationFrame(draw);
Why birdVelocity = flapStrength and not birdVelocity += flapStrength? Setting the velocity directly (with =) gives a consistent, snappy flap every time regardless of how fast the bird was already falling. Using += would make flapping feel inconsistent -- sometimes strong, sometimes weak.

Preventing the space bar from scrolling the page

There's one annoying problem: pressing Space normally scrolls the page down. We need to tell the browser "we handled it, don't do the default thing." 

document.addEventListener('keydown', function(event) {
  if (event.code === 'Space') {
    event.preventDefault();          // stop the page from scrolling
    birdVelocity = flapStrength;
  }
});

event.preventDefault() cancels the browser's default behavior for that key. Without it, your game and the page scroll at the same time -- very frustrating!

Try it yourself

  • Change flapStrength to -3 -- the flaps feel weak and the bird barely stays up
  • Change flapStrength to -12 -- each flap sends the bird rocketing upward
  • Change the event from 'Space' to 'ArrowUp' -- now the up arrow controls the bird
  • Add a second key listener for 'ArrowDown' that pushes the bird down (set velocity to 5)

Key takeaways

  • addEventListener('keydown', fn) lets you react to key presses
  • event.code identifies which key was pressed
  • Setting velocity to a negative number makes the bird jump upward
  • event.preventDefault() stops the browser's built-in key behavior
  • You now have all three ingredients: draw, update, input -- the complete game loop
