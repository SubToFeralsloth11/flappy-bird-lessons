Lesson 1: The Canvas & Drawing

The first thing every game needs: something on the screen.

Three ingredients of a browser game

Every page you see in a browser is made of three technologies:

HTML -- the skeleton. It declares what is on the page: text, images, buttons, and a special element for drawing called <canvas>.

CSS -- the skin. It controls how things look: colors, sizes, positions.

JavaScript -- the brain. It makes things happen: movement, clicks, game logic. It's a real programming language.

A game is just a web page that redraws itself 60 times per second. The <canvas> element is our playground. JavaScript paints onto it faster than the eye can see, and that creates the illusion of movement.

The <canvas> element

The canvas is an invisible grid of pixels. You tell JavaScript: "fill a rectangle at position (50, 100) with yellow" and it appears. Everything in our game will be drawn on this canvas.

Here's the smallest possible canvas you can create:

<canvas id="game" width="400" height="600"></canvas>

id="game" gives it a name so JavaScript can find it. width and height set the pixel dimensions.

And here's the JavaScript to draw a filled rectangle on it:

const canvas = document.getElementById('game');
const ctx = canvas.getContext('2d');
ctx.fillStyle = '#f1c40f';
ctx.fillRect(180, 280, 40, 40);

Let's unpack each line:

  • const canvas = ... -- grab the canvas element by its id
  • const ctx = ... -- get the "2D drawing context," our paintbrush
  • ctx.fillStyle = ... -- set the paint color (a hex code for gold/yellow)
  • ctx.fillRect(x, y, width, height) -- paint a filled rectangle

Coordinate system

The canvas uses a coordinate system where (0, 0) is the top-left corner. X increases to the right, Y increases downward. This is different from math class!

Try it yourself

Edit the sandbox code above and make these changes:

  • Change the bird's fill color to red (#e74c3c)
  • Move the bird to the far right side of the canvas
  • Add a second eye next to the first one
  • Try ctx.fillRect(0, 0, 200, 200) -- what happens? (Hint: it paints over the background)
Common mistake: Forgetting to run fillRect after setting fillStyle. The color applies to everything drawn after the assignment, not before. Think of it like dipping your brush -- you dip, then you paint.

Key takeaways

  • HTML gives you the <canvas>. JS paints on it.
  • getContext('2d') is your paintbrush.
  • Coordinates start at the top-left: (0,0). Y goes down.
  • fillRect(x, y, w, h) draws a filled rectangle.
  • Drawing order matters -- later rectangles paint on top of earlier ones.
